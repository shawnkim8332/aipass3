# Coding Principles

1. Variable Names
    1.1 use camelCase for identifier names (variables and functions).
    1.2 All names start with a letter.

2. Code Indentation
    2.1 Always use 4 spaces for indentation of code blocks

3. Complex Statement Rules
    3.1 Put the opening bracket at the end of the first line.
    3.2 Use one space before the opening bracket.
    3.3 Put the closing bracket on a new line, without leading spaces.
    3.4 Do not end a complex statement with a semicolon.

4. Line Length
    4.1 For readability, avoid lines longer than 80 characters.
    4.2 If a JavaScript statement does not fit on one line, the best place to break it, is after an operator or a comma.

5. Use Lower Case File Names

6. ID names should be defined in camelCase

7. class names should be defined in lowercase

8. no use of inline css styles and javascript

9. Html Comments should show the end of html div's
